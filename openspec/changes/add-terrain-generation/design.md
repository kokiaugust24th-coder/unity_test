# Design: 地形・マップ自動生成システム

## Context

- 参考品質: デスストランディング (侵食された岩肌・苔・河川)、エルデンリング (台地・崖・レガシーダンジョン周辺の地形演出)
- 既存基盤: add-open-world-core のセルストリーミング (256m グリッド)、ベイクパイプライン、HLOD
- ワークフロー方針: **生成 → エディタで手直し → 再生成しても手直しが消えない → ベイク**
- 想定規模: 2km〜8km 四方、高さ解像度 0.5〜1m/px

## Goals / Non-Goals

- Goals:
  - シードと設定アセットから**完全決定論的**に同じワールドを再現できる
  - 侵食シミュレーションを含む「地質学的に読める」地形 (尾根・谷・扇状地・崖)
  - バイオームによるテクスチャ・植生の一貫した支配
  - 道・川・POI をスプライン/マーカーで指定すると地形が自動追従 (整地・彫り込み)
  - 手動編集の保護。部分再生成で反復サイクルを短く
  - 生成物はそのまま既存ストリーミング/HLOD に乗る
- Non-Goals:
  - ランタイム無限生成 (別プロポーザル)
  - 洞窟・オーバーハング (ハイトマップの表現域外。メッシュスタンプで将来対応)
  - NavMesh ベイク、ゲームプレイ配置 (敵スポーン等)
  - 気候シミュレーションによる動的バイオーム変化

## Decisions

### D1: パイプライン構成 — 内製の固定ステージ合成パイプライン

```
HeightField 合成 (ノイズスタック)             ← または外部ハイトマップのインポート (補助)
  → 大規模構造 (大陸形状・台地・崖のマスク合成)
  → 侵食 (水力ドロップレット + 熱侵食)
  → Feature 彫り込み (道・川・POI 整地)      ← スプライン/マーカー入力
  → 手動差分レイヤー適用                     ← 手修正の再適用
  → バイオーム分類 (高度・傾斜・湿度/フロー)
  → スプラットマップ生成 (ルール評価)
  → 散布 (植生・岩)                          ← バイオーム/マスク参照
  → Terrain タイル書き出し + ベイク統合
```

- **内製する理由 (AI 駆動開発の経済性)**: 外部 GUI ツールは AI が操作できず自動化・再現性が
  パイプライン外に漏れる。内製ならシード→最終タイルまで全自動・決定論的で、
  チューニング自体を AI の反復対象にできる
- ノードグラフ化はしない。ステージ固定 + パラメータ化されたレイヤーリスト。
  各ステージは `IHeightFieldStage` で分離し差替え可能
- 補助入力として外部ハイトマップ/マスク (RAW16/EXR/PNG) のインポートも維持する

### D2: データモデル — TerrainGenerationSettings (ScriptableObject)

- ワールドサイズ、シード、ハイトマップ解像度 (m/px)、最大標高
- ノイズレイヤー配列 (種類: fBm / Ridged / Voronoi / Terrace、周波数・振幅・ブレンドモード・マスク)
- 侵食パラメータ (反復回数、雨量、堆積率、熱安息角)
- バイオーム定義配列 (名前、高度/傾斜/湿度範囲、TerrainLayer 群、散布ルール群)
- 散布ルール (プレハブ or DetailPrototype、密度、傾斜/高度範囲、ノイズマスク、最小間隔)
- Feature 設定 (道幅・法面勾配、川幅・深さ、POI 平坦化半径)

### D3: 中間データ — WorldHeightField (float[] + メタ)

- 全ステージはタイルではなく**ワールド全体の HeightField** 上で動作し、最後にタイル分割する
  (タイル境界をまたぐ侵食・道の連続性を保証)
- float 配列 + NativeArray、Burst Jobs で並列化。8k×8k (64MB) まで実用
- 湿度/フロー/バイオーム ID も同解像度のフィールドとして保持

### D4: 侵食 — 内製 (水力ドロップレット + 熱侵食)

- 水力: 粒子 (droplet) ベース。シードから粒子列を決定論的に生成。Jobs 並列時も
  粒子→タイルバケット分割で適用順序を固定し、バイト一致の再現性を保証
- 熱: 安息角超過分を隣接セルへ再分配する反復。崖の裾に自然な崖錐を形成
- 出力: 高さに加えフロー (水流集積) / 堆積フィールドを生成し、バイオーム・散布の入力にする
- **チューニング戦略 (最大リスクへの対策)**:
  - フィールド可視化デバッガ (高さ/フロー/堆積/傾斜のヒートマップ表示) を先に作る
  - パラメータプリセット (穏やかな丘陵 / 侵食渓谷 / 岩石高原 等) を仕様に含め、
    参照画像との見た目比較で反復する
  - ゴールデンハッシュテスト (決定論) + プレビュー解像度での高速反復
- 代替案 (却下): 外部ツール (Gaea 等) → GUI 手作業が必須で AI 駆動の自動パイプラインに
  組み込めない。パイプモデル GPU 侵食 → 品質は高いがデバッグ困難、将来ステージ差替えで対応

### D5: 地形基盤 — セル単位の Unity Terrain タイル

- 1 ストリーミングセル = 1 Terrain タイル (TerrainData アセット)。既定: 256m / 257px (約 1m/px)
- 隣接シームは `Terrain.SetNeighbors` + 境界頂点の完全一致 (タイル分割時に境界を共有) で解決
- Terrain タイルプレハブを Addressables 登録し、マニフェストの `terrainAddress` で管理
- 遠景: タイルごとに低ポリメッシュ (グリッド間引き + 高さサンプル) を生成し既存 HLOD 機構で表示

### D6: 手動編集の保護 — ロックマスク + 差分レイヤー

- **差分レイヤー**: 手動のハイトマップ編集は「生成結果との差分」として記録し、
  再生成後に再適用する (生成パラメータを変えても手直しが生き残る)
- **ロックマスク**: ペイントした領域は再生成で一切変更しない (POI 周辺の完全固定用)
- スプラット/散布の手動上書きも同様にマスクで保護

### D7: 道と川 — スプライン入力の彫り込み

- 入力は com.unity.splines のスプラインをシーンに配置 (RoadFeature / RiverFeature コンポーネント)
- 道: スプライン断面で整地 (中心フラット + smoothstep 法面)、最大縦断勾配の検証警告、
  スプラットを道マテリアルに上書き、散布を抑制するマスクを出力
- 川: 河道を彫り込み + 水面メッシュをスプラインに沿って生成。流量に応じた幅補間
- POI: マーカー (POIFeature) の半径内を指定高度に平坦化し、散布抑制マスクを出力

### D8: 散布の出力先の使い分け

- 草・小型ディテール → Terrain DetailPrototype (GPU インスタンシング、Terrain と共にストリーム)
- 木 → Terrain TreePrototype (ビルボード LOD が無料で付く)
- 大岩・遺跡等のプレハブ → **既存ベイクパイプラインの入力** (OpenWorldRegion 直下に生成配置
  → 通常のセルベイクで HLOD/レイヤー対応)。生成物には `GeneratedScatterTag` を付け、
  再散布時に生成分のみ削除して手動配置と区別する

### D9: 決定論

- すべての乱数は `Unity.Mathematics.Random` をシード + ステージ ID + セル座標から派生
- 同一入力 (settings + シード + Feature 配置) → バイト一致のハイトマップ (ハッシュでテスト)

### D10: コアとの境界 — セルコンテンツとして統合 (二重実装の禁止)

- terrain-streaming は独自のストリーミング判定・状態機械・キューを**持たない**。
  Terrain タイルは既存セル状態機械の「コンテンツ種別」の 1 つとして扱う
- 実装は `WorldManifest.CellEntry` の多コンテンツ化 (`contents: {kind, address}[]`) と、
  種別ごとの `ICellContentHandler` (Prefab / Terrain) の登録で行う。
  `SetNeighbors` 接続は Terrain ハンドラのアクティベートフック内で実行する
- 責務の線引き: **「何を作るか」= terrain 側 / 「いつロード・表示するか」= core 側**
- add-open-world-core は実装済みのため先にアーカイブし、本変更のマニフェスト拡張・
  コンテンツ抽象化は world-asset-pipeline / world-streaming への MODIFIED デルタへ
  変換する (アーカイブ完了までは ADDED として記述)

### D11: セル HLOD の一本化

- セルの HLOD スロットは 1 つのまま維持する。Terrain タイルの遠景メッシュは
  ベイク時にコンテンツ HLOD と**1 つの HLOD プレハブへ統合**し、既存 world-hlod の
  表示半径・スワップ規則をそのまま使う (HLOD 機構の二重化とアドレス競合を防ぐ)

## Risks / Trade-offs

- 侵食の計算時間 (8k² で数分) → Burst 並列 + 進捗バー + 解像度を落としたプレビューモード
- Terrain タイル間の法線/ライティング継ぎ目 → 境界 1px 共有 + SetNeighbors で緩和。残る場合は法線焼き込みを検討
- 差分レイヤーは生成結果が大きく変わると不自然になる → 差分適用後の急勾配を警告表示
- Terrain のドローコール → タイルは activation 圏のみロードなので実質 3x3〜5x5 枚
- メモリ: TreeInstance 大量配置 → セルあたり上限を設定し警告

## Migration Plan

新規追加のため移行なし。既存の手作り地形 (メッシュ床) と併用可能 (Terrain タイルは通常セルと同じ枠組み)。ロールバックは `Generated/Terrain/` の削除で完結。
