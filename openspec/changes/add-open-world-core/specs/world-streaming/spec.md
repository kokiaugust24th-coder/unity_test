# world-streaming Spec Delta

## ADDED Requirements

### Requirement: グリッドセル分割

システムはワールドを XZ 平面の均一グリッドセル(サイズは `WorldPartitionConfig` で設定、既定 256m)に分割して管理しなければならない (SHALL)。セル境界を跨ぐオブジェクトまたは明示指定されたオブジェクトは Always Loaded セルに分類されなければならない (SHALL)。

#### Scenario: 座標からセルへの解決
- **WHEN** ワールド座標 (300, 0, -100) をセルサイズ 256m で解決する
- **THEN** セル座標 (1, -1) が返される

#### Scenario: 境界跨ぎオブジェクトの分類
- **WHEN** バウンズが複数セルに跨るオブジェクトをベイクする
- **THEN** そのオブジェクトは Always Loaded セルに分類される

### Requirement: セル状態機械

各セルは `Unloaded → Loading → Loaded → Activated` の状態を持たなければならない (SHALL)。`Loaded` はアセットがメモリ上にあり非表示、`Activated` はワールドに表示・有効化された状態とする。状態遷移は必ず隣接状態間でのみ行われなければならない (SHALL)。

#### Scenario: 段階的アクティベーション
- **WHEN** ストリーミングソースがロード半径内かつアクティベート半径外にあるセルを評価する
- **THEN** セルは `Loaded` まで遷移し `Activated` にはならない

#### Scenario: アクティベート
- **WHEN** ソースがアクティベート半径内に入る
- **THEN** セルは `Activated` に遷移しオブジェクトが表示・有効化される

### Requirement: ストリーミングソース抽象

システムは `IStreamingSource`(位置・半径倍率・優先度)を複数登録できなければならない (SHALL)。セルの要求状態は全ソースの評価結果の最大値で決定されなければならない (SHALL)。これは将来のマルチプレイヤー(サーバー上の複数プレイヤー)およびカメラ先読みへの拡張点である。

#### Scenario: 複数ソースの合成
- **WHEN** ソース A がセルを `Loaded`、ソース B が同セルを `Activated` と要求する
- **THEN** セルの要求状態は `Activated` になる

#### Scenario: ソースの動的登録解除
- **WHEN** 登録済みソースが解除される
- **THEN** 次回評価から当該ソースは無視され、不要になったセルはアンロード対象になる

### Requirement: 距離判定とヒステリシス

セルとソースの距離判定はセル AABB の最近点距離を用いなければならない (SHALL)。ロード境界とアンロード境界にはヒステリシス(既定 32m、設定可能)を設け、境界付近での状態振動を防がなければならない (SHALL)。

#### Scenario: 境界振動の防止
- **WHEN** ソースがロード半径の境界を小刻みに往復する
- **THEN** セルはヒステリシス幅を超えるまでアンロードされない

### Requirement: 非同期ロードとフレーム予算

セルのロードは Addressables による非同期処理で行い、メインスレッドのインスタンス化処理は 1 フレームあたりの時間予算(既定 2ms、設定可能)を超えてはならない (MUST)。同時 in-flight ロード数は上限(既定 4)を持たなければならない (SHALL)。ロードキューは距離昇順かつソース優先度順で処理されなければならない (SHALL)。

#### Scenario: フレーム予算の遵守
- **WHEN** 1 フレーム内で複数セルのインスタンス化が予算を超過しそうになる
- **THEN** 残りの処理は次フレーム以降に繰り越される

#### Scenario: 近いセルの優先
- **WHEN** 複数セルが同時にロード要求される
- **THEN** ソースに近いセルから順にロードされる

### Requirement: アンロードと資源解放

`Unloaded` への遷移時、システムはセルインスタンスを破棄し Addressables ハンドルを Release してメモリを解放しなければならない (SHALL)。ロード中のセルがアンロード対象になった場合、ロード完了を待ってから破棄しなければならない (SHALL)(ハンドルリーク防止)。

#### Scenario: ロード中キャンセル
- **WHEN** `Loading` 状態のセルがソース圏外になる
- **THEN** ロード完了後にインスタンス化せず即座にハンドルが Release される
