# world-asset-pipeline Spec Delta

## ADDED Requirements

### Requirement: ワールドベイク

エディタは、オーサリングシーンの静的オブジェクトをセルごとに分類し、セルプレハブとして `Assets/OpenWorld/Generated/Cells/` に生成して Addressables に登録するベイク処理を提供しなければならない (SHALL)。ベイクは冪等でなければならず (MUST)、オーサリングシーンを変更してはならない (MUST)。生成物の削除のみで完全にロールバックできなければならない (SHALL)。

#### Scenario: ベイクの実行
- **WHEN** オーサリングシーンに対しベイクを実行する
- **THEN** セルごとのプレハブが生成・Addressables 登録され、オーサリングシーンは無変更のまま残る

#### Scenario: 冪等性
- **WHEN** 変更のないオーサリングシーンに対しベイクを 2 回実行する
- **THEN** 2 回目の成果物は 1 回目と同一で、Addressables 登録の重複は発生しない

### Requirement: 差分ベイク

ベイクパイプラインはセル単位のダーティ判定(内容ハッシュ)を行い、変更されたセルのみ再生成する差分モードを提供しなければならない (SHALL)。全体リベイクも明示的に実行可能でなければならない (SHALL)。

#### Scenario: 変更セルのみ再ベイク
- **WHEN** 1 セル内のオブジェクトのみ移動して差分ベイクを実行する
- **THEN** 当該セル(移動元・移動先)のプレハブのみ再生成される

### Requirement: ベイク検証と警告

ベイク処理は次の検証を行い、問題を警告として報告しなければならない (SHALL): プレハブ階層の過剰な深さ、非静的コンポーネントの混入、Addressables 依存の重複(共有アセットの複数バンドル混入)。エラー時はベイクを中断し、部分的な成果物を残してはならない (MUST)。

#### Scenario: 依存重複の検出
- **WHEN** 同一マテリアルが複数セルバンドルに重複して含まれる状態でベイクする
- **THEN** 重複が警告として報告され、共有グループへの移動が提案される

#### Scenario: エラー時の中断
- **WHEN** ベイク途中でアセット生成に失敗する
- **THEN** 当該ベイクの成果物はすべて破棄され、直前の状態が維持される

### Requirement: メモリ管理

ランタイムは Addressables ハンドルとセルインスタンスの対応を追跡し、リークを検出可能にしなければならない (SHALL)。全セルアンロード後、セル由来のバンドルメモリは解放されなければならない (SHALL)。

#### Scenario: リークなしの解放
- **WHEN** 全ストリーミングソースを削除して全セルがアンロードされる
- **THEN** 未解放の Addressables ハンドルが 0 件であることが統計 API で確認できる
